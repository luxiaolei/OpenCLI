# ChatGPT Pro 图片能力冻结（/images 实测）

- 冻结时间：2026-04-11（Asia/Shanghai）
- 账号态：已登录，页面可见 `Pro`
- 检查方式：`opencli browser open https://chatgpt.com/images` + DOM/state 实测
- 范围：只记录**当前真实可见**的图片页能力与最小结果态；不把猜测写成事实

## 1. 结论先行

当前 Pro 账号下，`/images` 页是一个**以 prompt + 上传图片 + 预置风格/任务卡片**为核心的图片工作台。

实测可确认：

1. **没有看到独立的模型选择器**（如 4o / gpt-image-1 / 质量档位）。
2. **没有看到独立的比例、尺寸、质量、张数、seed、变体数**等参数控件。
3. **真实存在图片上传/参考图入口**，且不止一个：有拖拽上传、普通上传、照片、相机几个输入位。
4. **“风格”当前是通过预置卡片暴露，不是通用下拉/单选控件。**
5. 最小生成后，在结果卡片层可见的动作是：**打开图片、编辑图片、分享此图片**；**未在当前页卡片层看到“下载”“生成变体”独立按钮**。

## 2. /images 落地页真实可见面

### 2.1 顶层页身份

实测页面信息：

- URL：`https://chatgpt.com/images/`
- Title：`ChatGPT 图片 | AI 图片生成器`
- 账号区域可见：`hzcodex` + `Pro`

### 2.2 主创作区控件（真实 UI）

实测可见/可确认的主控件：

- 一个图片 prompt 输入框
  - placeholder：`描述新图片`
- 一个添加入口按钮
  - aria-label：`添加文件等`
- 一个听写按钮
  - aria-label：`开始听写`
- 一个发送按钮
  - aria-label：`发送提示`
- 拖拽上传区域
  - 文案：`拖放图片以上传`

### 2.3 真实存在的上传/参考图输入位

DOM 中可确认存在以下 file input：

- `image-gen-action-modal-upload-photos`
- `image-gen-action-modal-upload-camera`
- `images-app-drop-container-input`
- `upload-files`
- `upload-photos`
- `upload-camera`

可接受格式（实测 DOM）：

- 拖拽区：`png / jpg / jpeg / mpo / gif / webp / heic / heif`
- composer 上传：`jpg / jpeg / mpo / png / webp / gif`
- photos / camera：`image/*`

**冻结判断：**

- “参考图/上传图生图/带图编辑”是**真实 UI 能力**，不是只靠 prompt 想象出来的能力。
- 至少在入口层，系统明确支持：**本地图片上传、照片、相机、拖拽上传**。

### 3.2 实际 UI：任务预置卡片

在 `发现新事物` 区块，实测看到了这些**真实按钮/卡片**：

**冻结判断：**

- 这些不是“底层参数控件”，而是**任务模板/提示模板入口**；
- 它们代表一些可复用的高频图片工作流，但当前看起来仍更像**预制 prompt 模板**，不是独立算法开关。

### 3.3 没看到的独立 UI 参数

在当前实测页里，**没有看到**以下独立参数控件：

- 模型选择器（图片专用）
- 质量（如 standard / hd）
- 比例（如 1:1 / 16:9 / 9:16）
- 尺寸
- 一次生成张数
- seed
- 变体数量
- 背景透明开关
- 风格强度滑杆

**冻结判断：**

这些能力即使可能能通过 prompt 间接表达，**当前也不是独立 UI 控件**。

## 4. 最小生成测试：结果态真实可见动作

### 4.1 测试方法

做了一个 harmless 最小测试：

- prompt：`一只蓝色马克杯，纯白背景，产品图`

### 4.2 结果态实测

生成完成后，页面/结果卡片层实测可见：

- `打开图片：蓝色陶瓷杯`
- `编辑图片`
- `分享此图片`

并且页面正文中可见 `编辑` 文案。

### 4.3 结果态冻结判断

**已确认存在的结果动作：**

- 打开图片
- 编辑图片
- 分享图片

**当前未在结果卡片层看到的动作：**

- 下载图片
- 生成变体 / variations
- 重新生成
- 放大 / upscale

> 说明：这里只冻结“当前页结果卡片层真实看见的动作”。
> 是否在“点开图片后的查看器/详情层”还会出现 `下载`，本次没有稳定确认，因此不写成事实。

## 5. “最高级”图片能力或模式名称：当前可见面判断

这是本次最关键的冻结点之一。

### 5.1 明确可见的最高级账号/权益标识

实测明确可见：

- 账号带 `Pro` 标识

### 5.2 没有看到的内容

在 `/images` 页面上，**没有看到**明确的图片模型/模式名称，例如：

- `4o image`
- `gpt-image-1`
- `GPT-4o image generation`
- 单独的“高质量模式 / 专业模式 / RAW 模式”名称

### 5.3 冻结结论

**当前账号可见面的“最高级图片能力”不是以模型名暴露，而是以 `Pro` 权益 + `/images` 专用图片工作台暴露。**

也就是说：

- **最高可见权益名：`Pro`**
- **最高可见产品/工作台名：`图片` / `ChatGPT 图片 | AI 图片生成器`**
- **未见到更细的图片模型名或高级图片模式名**

这点对后续命令设计很重要：**不要先假定 UI 已经公开了某个具体图片模型名。**

## 7. 对未来 `chatgpt image` 命令的建议命令面

基于本次冻结，建议未来命令面**先贴真实 UI**，不要超前设计一堆未验证参数。

### 7.1 建议优先支持（MVP）

#### 1) 纯生成

```bash
chatgpt image create --prompt "一只蓝色马克杯，纯白背景，产品图"
```

#### 2) 带参考图 / 上传图

```bash
chatgpt image create --prompt "把它做成产品海报" --image ./ref.jpg
chatgpt image create --prompt "保留主体，换成霓虹幻想风格" --image ./a.png --image ./b.png
```

#### 3) 风格预置卡

```bash
chatgpt image create --prompt "人物肖像" --preset "霓虹幻想"
chatgpt image create --prompt "宠物照片" --preset "影楼肖像"
```

> 这里建议叫 `--preset` 或 `--style-card`，不要先叫 `--style-mode`。

#### 4) 任务模板卡

```bash
chatgpt image create --template "移除背景中的人物" --image ./street.jpg
chatgpt image create --template "为我换装" --image ./portrait.jpg
chatgpt image create --template "重新装饰我的房间" --image ./room.jpg
```

#### 5) 列出我的图片 / 打开结果

```bash
chatgpt image list
chatgpt image open --latest
chatgpt image open --id <image-or-thread-id>
```

#### 6) 编辑已生成图片

```bash
chatgpt image edit --latest --prompt "把杯子改成磨砂黑色"
chatgpt image edit --id <image-id> --prompt "把背景改成暖灰色影棚"
```

#### 7) 分享（仅当用户明确允许）

```bash
chatgpt image share --latest
```

### 7.2 暂时不要提前承诺的参数

在再次验证前，建议**不要**把下面这些做成正式稳定参数：

```bash
--model
--quality
--aspect-ratio
--size
--n
--seed
--variant
--upscale
--download
```

原因：本次冻结里，这些都**没有在当前 UI 上被稳定看到**。

### 7.3 更稳的抽象方式

建议把命令面分成三层：

1. **create**：文生图 / 图生图
2. **preset/template**：调用页面现成风格卡、任务卡
3. **asset actions**：open / edit / share / list

这样最贴近当前真实产品面，也最不容易因为 UI 漂移而失效。

---

## 8. 本次冻结的关键结论

1. `/images` 页当前是 **prompt + 上传 + 预置卡片** 的图片工作台，不是参数面板型工作台。
2. **上传/参考图/相机/照片** 是实打实的 UI 能力。
3. **风格能力有 UI**，但形态是**风格卡片**，不是通用风格参数控件。
4. 最小生成后，当前页结果层可见 **打开图片 / 编辑 / 分享**，但**未见下载/变体**独立按钮。
5. 对 Pro 用户来说，**当前最明确可见的高级能力标识是 `Pro` 权益本身**，而不是某个明确写出来的图片模型名。
